# Tech Stack: rmotis

## Language & Core
- **R:** Primary language for the package.
- **motis.client:** Autogenerated R client for the MOTIS API (OpenAPI 3).
- **httr2:** For making robust HTTP requests to the MOTIS server.
- **processx / ps:** For managing the MOTIS server background process and monitoring system resources.
- **RcppSimdJson:** For ultra-fast JSON parsing of large MOTIS responses.

## Data & Spatial
- **tidyverse (dplyr, purrr, tibble, tidyr, rlang):** For data manipulation and functional programming patterns.
- **sf:** For handling spatial data (Simple Features).
- **googlePolylines:** For decoding polyline-encoded geometries from MOTIS.
- **glue:** For clean string interpolation.

## UI & Visualization
- **Shiny / shinyjs:** For the interactive routing and debugging GUI.
- **mapgl:** For high-performance Mapbox GL/MapLibre GL visualizations.
- **DT:** For interactive data tables in the GUI.

## Infrastructure & Quality
- **testthat:** For unit and integration testing.
- **roxygen2:** For documentation and NAMESPACE management.
- **GitHub Actions:** For CI/CD (R-CMD-check, code coverage, pkgdown).
